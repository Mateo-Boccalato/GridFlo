Overview

Programs as connected grids

GridFlo is a small programming language interpreter written in Python. Programs are drawn as two-dimensional dataflow grids in .gf files, then parsed and executed as graphs. 

name:value >──[trim]──[upper]──< result:value

Data enters through input ports, moves along wires, passes through transform cells, and exits through output ports.

Project status: GridFlo is an experimental interpreter and language prototype for exploring 2D programming ideas.

Quick Start

No third-party dependencies

Python 3.9 or newer is recommended. Check your Python version first:

python --version

Run the hello example:

python gridflow.py run "sample programs/hello.gf"
output = 'GRIDFLO'

Run FizzBuzz:

python gridflow.py run "sample programs/fizzbuzz.gf" n=15
answer = ['1', '2', 'Fizz', '4', 'Buzz', 'Fizz', '7', '8', 'Fizz', 'Buzz', '11', 'Fizz', '13', '14', 'FizzBuzz']

Command Reference

Run, check, and demo

Run a program

python gridflow.py run <file.gf> [key=value ...]

Input values are passed as key=value pairs. The CLI tries to convert numeric inputs to int or float; otherwise the value is passed as a string.

Check a program

python gridflow.py check <file.gf>

This lexes, parses, wires, and type-checks the program without executing it.

Run demos and syntax checks

python gridflow.py demo
python -m py_compile gridflow.py lexer.py parser.py interpreter.py

Language Reference

Cells, wires, types, and transforms

Core idea

input:value >──[transform]──< output:value

Each cell has a position in the 2D source grid. The parser uses those positions to trace wire characters and build a directed graph.

Supported wire types

  • value: a single value, such as a number, string, or boolean.
  • stream: a sequence/list of values, such as [1, 2, 3].
  • signal: a boolean condition, usually produced by transforms like [?=0].
  • bag: a collected group of values, usually created by {collect}.

Ports and constants

name:value >
items:stream >
flag:signal >
values:bag >

< result:value

*42
*"hello world"
*true
*false

Wires

Horizontal: ── or --
Vertical:   │ or |
Corners:    ┐ ┘ └ ┌
Junctions:  ┤ ├ ┬ ┴ ┼ +
Taps:       · or .

Use monospace editing and keep alignment exact. GridFlo relies on character positions to trace connections.

Transform cells

[trim] [upper] [lower] [str] [len] [reverse] [split] [!]
[int] [float]
[+1] [-5] [×2] [*2] [÷3] [/3] [%2] [^2]
[+] [-] [×] [*] [÷] [/] [%] [^]

Comparisons and signals

[=5] [==5] [!=0] [≠0] [>3] [<10] [>=60] [≤100]
[?=0] [?!=0] [?>5] [?<10] [?>=60] [?<=100]

Gates

The ? cell routes based on a signal input. True goes right; false goes down.

n:value >──[%3]──[?=0]──?──[const:Fizz]──< out:value
                         │
                         [str]───────────< out:value

Streams, collect, and reduce

n:value >──[range]──[map:double]──< out:stream

a:value >──────────────┐
                        {collect}──(avg)──< average:value
b:value >──────────────┘

Reducers include (sum), (max), (min), (count), (avg), (join), and (product).

Plates

plate shout:
text:value >──[trim]──[upper]──[!]──< out:value
end

message:value >──[shout]──< result:value

Architecture

Lexer to parser to interpreter

source text -> lexer -> parser -> interpreter
01

Lexer

Scans source text into positioned tokens, including wire characters.

02

Parser

Builds a spatial index, traces wires, creates cells, and validates types.

03

Interpreter

Executes the graph, fires ready cells, routes gates, and collects outputs.

Troubleshooting

Common issues

Inputs: [] when you expected an input

Make sure the input port has a > marker.

Correct:   n:value >
Incorrect: n:value

A wire does not connect

  • Horizontal wires should start at the cell's right edge.
  • Vertical wires should align under the source cell or branch point.
  • Corners should line up exactly with the path.
  • Edit in a monospace font.
python gridflow.py run file.gf --verbose

A gate does not take the expected branch

Make sure the gate receives a signal. Signal-producing transforms start with ?, for example [?=0].

[map:<plate>] fails during check

Make sure the plate exists and the spelling matches exactly.

scalar type errors

Use value instead of scalar in new .gf files.

Current limitations

  • The language is experimental and intentionally small.
  • Wire tracing depends on exact character positions.
  • There is no package manager or module system.
  • The CLI prints Python-style representations of output values.
  • Error messages are basic and may point to graph rows rather than full source spans.
